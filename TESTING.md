file changes)\n```bash\nnpm run test:watch\n```\n\n#### Run tests with coverage report\n```bash\nnpm run test:coverage\n```\n\n## Test Structure\n\nAll test files are located in `src/__tests__/` and Functions\n- Lines\n- Statements\n\nRun coverage report:\n```bash\nnpm run test:coverage\n```\n\nCoverage reports are generated in the `coverage/` directory.\n\n## Troubleshooting\n\n### Tests not running?\n1. Ensure all dependencies are installed: `npm install`\n2. Check Node.js version compatibility (Node 18+)\n3. Verify TypeScript files compile: `npm run compile-only`\n\n### Type errors in tests?\n- Make sure `@types/jest` is installed\n- Run `npm install` to update all dependencies\n- Verify `tsconfig.json` includes test files\n\n### Import errors?\n- Check file paths in imports match your project structure\n- Ensure `.js` extensions in ES module imports\n- Verify `jest.config.js` module name mapping\n\n### Coverage not collecting?\n- Run: `npm run test:coverage`\n- Coverage reports appear in `coverage/` directory\n- Check `jest.config.js` for coverage configuration\n\n## Writing New Tests\n\nWhen adding new tests:\n\n1. Create test file in appropriate `src/__tests__/` subdirectory\n2. Follow naming convention: `ClassName.test.ts`\n3. Use Jest's `describe()` and `it()` blocks\n4. Include `beforeEach()` for setup when needed\n5. Test both happy paths and edge cases\n\n### Test Template\n```typescript\nimport YourClass from '../../path/to/Class';\n\ndescribe('YourClass', () => {\n  let instance: YourClass;\n\n  beforeEach(() => {\n    instance = new YourClass();\n  });\n\n  describe('method name', () => {\n    it('should do something', () => {\n      expect(instance).toBeDefined();\n    });\n  });\n});\n```\n\n## GitHub Actions\n\nTests run automatically on:\n- Every push to `unit_tests` or `main` branch\n- Every pull request to `unit_tests` or `main`\n- Node.js 20.x and 21.x\n- Coverage uploaded to Codecov\n\nCheck workflow status in the Actions tab of your repository.\n\n## Best Practices\n\n✅ **Do:**\n- Test one thing per test\n- Use descriptive test names\n- Keep tests independent\n- Clean up resources in afterEach\n- Test edge cases\n\n❌ **Don't:**\n- Create dependencies between tests\n- Test implementation details\n- Use hardcoded timeouts\n- Ignore test failures\n- Skip important test cases\n\n## Performance Tips\n\n- Use `test:watch` during development for fast feedback\n- Run specific tests while working on a feature\n- Use `--bail` to stop on first failure: `npm test -- --bail`\n- Parallel test execution is enabled by default\n\n## CI/CD Integration\n\nAll tests must pass before merging to `main`. GitHub Actions will:\n1. Install dependencies\n2. Run all tests\n3. Collect coverage\n4. Upload to Codecov\n5. Report results\n
